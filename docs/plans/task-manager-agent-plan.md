## Summary

A new **TaskManager** agent providing workspace-scoped project and task management with DAG-like dependencies. The data model is **Workspace ‚Üí Project ‚Üí Task** with `dependsOn[]` for DAG edges and `parentTaskId` for subtask hierarchy. Tasks bidirectionally link to vault notes and auto-load when a workspace is loaded. Storage follows the existing JSONL + SQLite hybrid pattern. 10 tools exposed via the Two-Tool Architecture.

#### Design Approach

**Component Architecture:**
```
TaskManagerAgent (extends BaseAgent)
  ‚îú‚îÄ‚îÄ tools/projects/ (4 tools: create, list, update, archive)
  ‚îú‚îÄ‚îÄ tools/tasks/ (4 tools: create, list, update, move)
  ‚îú‚îÄ‚îÄ tools/tasks/queryTasks.ts (DAG queries consolidated)
  ‚îî‚îÄ‚îÄ tools/links/linkNote.ts (bidirectional note linking)

Services:
  TaskService (business logic facade)
    ‚îú‚îÄ‚îÄ ProjectRepository (extends BaseRepository)
    ‚îú‚îÄ‚îÄ TaskRepository (extends BaseRepository)
    ‚îî‚îÄ‚îÄ DAGService (pure computation, no storage)
```

**Key Design Principles:**
1. **DAGService is pure** ‚Äî takes arrays of tasks/edges, returns computed results. No storage access. Maximally testable.
2. **TaskService is the facade** ‚Äî orchestrates repositories + DAGService. Enforces business rules (workspace scoping, status transitions, cycle prevention).
3. **Tools are thin** ‚Äî validate params ‚Üí call service ‚Üí return result. No business logic in tools.
4. **Repositories own storage** ‚Äî JSONL events + SQLite cache, following BaseRepository pattern.

#### Key Decisions

| Decision | Options | Recommendation | Rationale |
|----------|---------|----------------|-----------|
| DAG storage | A) Adjacency list + separate table, B) Closure table, C) Inline JSON array | **A** | Recursive CTEs handle transitive queries; closure tables add O(N¬≤) maintenance; inline JSON prevents JOINs |
| Cycle detection location | A) Repository (SQL), B) Service (TypeScript), C) Pure function | **C** ‚Äî in DAGService | Pure functions = 100% testable without mocks; DAGService is stateless |
| Project entity | A) First-class entity with table, B) Lightweight tags | **A** | Projects have their own lifecycle (status, metadata); tags don't support this |
| Repositories | A) Single combined, B) Separate Project + Task repos | **B** | Different schemas, lifecycles, query patterns; SRP |
| JSONL file organization | A) Per-workspace, B) Per-project, C) Per-task | **A** ‚Äî `tasks_[workspaceId].jsonl` | Matches existing `ws_[id].jsonl` pattern; per-project fragments too much |
| Note links storage | A) JSON column on task, B) Separate junction table | **B** ‚Äî `task_note_links` | Enables bidirectional queries (task‚Üínotes AND note‚Üítasks) without full-table scans |
| `blocked` status | A) Stored and auto-updated, B) Derived/computed in queries | **B** ‚Äî Derived | Avoids state sync issues between dependency changes and task status; computed via subquery |
| Parent task auto-complete | A) Auto-complete when all children done, B) Independent | **B** ‚Äî Independent | Parent status is independent of children; auto-complete creates unexpected behavior |
| Delete project behavior | A) CASCADE delete tasks, B) Error if tasks exist | **A** ‚Äî CASCADE | FK ON DELETE CASCADE handles it; JSONL preserves history for recovery |
| Delete task with dependents | A) Remove from dependents' edges, B) Error | **A** ‚Äî CASCADE cleanup | FK ON DELETE CASCADE removes edges automatically; dependents become unblocked |
| Cross-workspace references | A) Allow, B) Prohibit | **B** ‚Äî Prohibit | Workspace isolation is a core architectural principle |
| Sub-projects | A) Support nesting, B) Keep flat | **B** ‚Äî Flat for v1 | YAGNI; can add later if needed |

## Data Model

### SQLite Schema (4 tables, 13 indexes)

```sql
-- PROJECTS
CREATE TABLE IF NOT EXISTS projects (
  id TEXT PRIMARY KEY,
  workspaceId TEXT NOT NULL,
  name TEXT NOT NULL,
  description TEXT,
  status TEXT NOT NULL DEFAULT 'active',    -- 'active', 'completed', 'archived'
  created INTEGER NOT NULL,
  updated INTEGER NOT NULL,
  metadataJson TEXT,
  FOREIGN KEY(workspaceId) REFERENCES workspaces(id) ON DELETE CASCADE,
  UNIQUE(workspaceId, name)
);

-- TASKS
CREATE TABLE IF NOT EXISTS tasks (
  id TEXT PRIMARY KEY,
  projectId TEXT NOT NULL,
  workspaceId TEXT NOT NULL,                -- denormalized for fast workspace queries
  parentTaskId TEXT,                         -- subtask hierarchy (NULL = top-level)
  title TEXT NOT NULL,
  description TEXT,
  status TEXT NOT NULL DEFAULT 'todo',       -- 'todo', 'in_progress', 'done', 'cancelled'
  priority TEXT DEFAULT 'medium',            -- 'critical', 'high', 'medium', 'low'
  created INTEGER NOT NULL,
  updated INTEGER NOT NULL,
  completedAt INTEGER,
  dueDate INTEGER,
  assignee TEXT,
  tagsJson TEXT,                             -- JSON array of string tags
  metadataJson TEXT,
  FOREIGN KEY(projectId) REFERENCES projects(id) ON DELETE CASCADE,
  FOREIGN KEY(workspaceId) REFERENCES workspaces(id) ON DELETE CASCADE,
  FOREIGN KEY(parentTaskId) REFERENCES tasks(id) ON DELETE SET NULL
);

-- TASK DEPENDENCIES (DAG edges)
CREATE TABLE IF NOT EXISTS task_dependencies (
  taskId TEXT NOT NULL,                      -- the task that depends on another
  dependsOnTaskId TEXT NOT NULL,             -- the task that must complete first
  created INTEGER NOT NULL,
  PRIMARY KEY(taskId, dependsOnTaskId),
  FOREIGN KEY(taskId) REFERENCES tasks(id) ON DELETE CASCADE,
  FOREIGN KEY(dependsOnTaskId) REFERENCES tasks(id) ON DELETE CASCADE
);

-- TASK-NOTE LINKS (bidirectional)
CREATE TABLE IF NOT EXISTS task_note_links (
  taskId TEXT NOT NULL,
  notePath TEXT NOT NULL,
  linkType TEXT NOT NULL DEFAULT 'reference', -- 'reference', 'output', 'input'
  created INTEGER NOT NULL,
  PRIMARY KEY(taskId, notePath),
  FOREIGN KEY(taskId) REFERENCES tasks(id) ON DELETE CASCADE
);
```

### Key DAG Queries

**Cycle detection (before inserting edge):**
```sql
WITH RECURSIVE ancestors(id) AS (
  SELECT dependsOnTaskId FROM task_dependencies WHERE taskId = :newDependsOnId
  UNION ALL
  SELECT td.dependsOnTaskId FROM task_dependencies td
  JOIN ancestors a ON a.id = td.taskId
)
SELECT EXISTS(SELECT 1 FROM ancestors WHERE id = :newTaskId) AS hasCycle;
```

**Next actionable tasks (ready to work on):**
```sql
SELECT t.* FROM tasks t
WHERE t.projectId = ? AND t.status = 'todo'
  AND NOT EXISTS (
    SELECT 1 FROM task_dependencies td
    JOIN tasks dep ON dep.id = td.dependsOnTaskId
    WHERE td.taskId = t.id AND dep.status NOT IN ('done', 'cancelled')
  )
ORDER BY t.priority DESC, t.created ASC;
```

**Topological sort (execution order):**
```sql
WITH RECURSIVE topo(id, depth) AS (
  SELECT t.id, 0 FROM tasks t WHERE t.projectId = ?
    AND NOT EXISTS (SELECT 1 FROM task_dependencies td WHERE td.taskId = t.id)
  UNION ALL
  SELECT t.id, topo.depth + 1 FROM tasks t
  JOIN task_dependencies td ON td.taskId = t.id
  JOIN topo ON topo.id = td.dependsOnTaskId WHERE t.projectId = ?
)
SELECT id, MAX(depth) as execOrder FROM topo GROUP BY id ORDER BY execOrder;
```
