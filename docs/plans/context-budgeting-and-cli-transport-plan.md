## Summary

This plan originally combined two distinct workstreams:

1. **Transport failure before inference starts**
   Local CLI providers currently send large prompt payloads through command-line argv, which can fail on Windows with `spawn ENAMETOOLONG` before the model ever sees the request.

2. **Context overrun without uniform pre-send handling**
   The codebase already has context estimation, message compaction, and LLM-driven state preservation, but automatic pre-send compaction is effectively limited to `webllm`.

That split turned out to be cleaner operationally and architecturally, so this combined plan has been replaced by two narrower plans:

- `docs/plans/claude-code-connect-silent-failure-plan.md`
  Covers Claude Code and Gemini CLI transport hardening only.

- `docs/plans/context-compaction-engineering-plan.md`
  Covers app-level budgeting, durable/hierarchical compaction, transcript references, and exact recovery flow.

#### Design Approach

Introduce a **two-layer control plane** for context safety:

1. **Transport Safety Layer**
   Ensures the request can be physically sent to the provider process or endpoint.
   - Local CLI providers must avoid sending large prompt payloads through argv.
   - Transport-specific budget checks still remain as a backstop even after transport changes.

2. **Context Budget Layer**
   Runs before every send and determines whether the conversation is safe to send raw, should warn, should compact first, or should block.
   - Uses real usage when available.
   - Falls back to normalized estimation otherwise.
   - Applies a provider/model-specific **effective compaction cap**, which may be lower than the model’s advertised context window.

#### Core Decision: Soft Cap vs True Model Context

The repo’s model metadata already advertises context windows such as:
- Claude Code: `200k`
- Codex: `200k`, `400k`, `1.05M`
- Gemini CLI: `1.048M`
- GitHub Copilot: mixed, up to `1.05M`

Those values should remain as model metadata. However, for initial operational safety, compaction policy should use:

```typescript
effectiveCompactionCap = min(model.contextWindow, providerSoftCap ?? Infinity)
```

For phase 1, set:

```typescript
providerSoftCap = 200_000
```

for:
- `anthropic-claude-code`
- `google-gemini-cli`
- `openai-codex`
- `github-copilot`

This gives one conservative ceiling while still preserving correct model metadata for display and future tuning.

#### Key Decisions

| Decision | Options | Resolution | Rationale |
|----------|---------|------------|-----------|
| Local CLI transport | Large argv vs stdin/files | **stdin/files** | Removes Windows `ENAMETOOLONG` failure mode cleanly |
| Compaction eligibility | `webllm` only vs all selected providers | **Expand to Claude Code, Gemini CLI, Codex, Copilot** | These providers can still receive oversized payloads or overlong conversations |
| Budget cap | True model window only vs lower safety cap | **Lower safety cap** | Gives predictable behavior for opaque providers and stateless integrations |
| Initial soft cap | Per-provider custom values vs one conservative value | **200k shared cap** | Pragmatic rollout with low policy complexity |
| Usage accounting | Provider-native raw shapes vs canonical internal shape | **Canonical internal shape** | Fixes current mismatch and simplifies budgeting |
| Compaction storage | In-memory only vs conversation metadata | **Conversation metadata** | Prevents summary loss after reload/restart |
| Native provider compaction | Trust provider-specific behavior vs app-level compaction | **App-level compaction first** | Provider-native compaction is opaque or unobservable in current adapters |

#### Implementation Sequence

**Wave 1 — Fix the original bug and unify usage accounting**
1. Add canonical internal token usage shape and update `TokenCalculator` to read both camelCase and snake_case.
2. Ensure `StreamingResponseService` persists usage in a stable canonical shape.
3. Fix Claude Code adapter transport using file-backed system prompt + stdin prompt.
4. Map Windows spawn length failures to actionable provider errors.
5. Improve user-visible error messages so provider-specific failures are surfaced instead of generic “Failed to send message”.

**Wave 2 — Introduce global context budgeting**
6. Add `ContextBudgetService` with provider policies and action thresholds.
7. Compute effective compaction cap using `min(model.contextWindow, providerSoftCap)`.
8. Seed phase-1 policy with `200k` soft cap for Claude Code, Gemini CLI, Codex, and GitHub Copilot.
9. Integrate preflight budgeting into `ChatView` before every send.

**Wave 3 — Expand auto-compaction**
10. Replace `webllm`-only gating in `ModelAgentManager` with policy-driven eligibility.
11. Route Claude Code, Gemini CLI, Codex, and GitHub Copilot through the same compaction decision path.
12. Keep `ContextPreservationService` as the preferred first attempt where tool execution is viable; fall back to heuristic truncation.

**Wave 4 — Persist compaction state**
13. Add durable compaction record(s) to conversation metadata.
14. Restore `previous_context` from metadata on conversation load.
15. Update retries/reloads/branching to preserve compaction summary continuity.

**Wave 5 — Gemini/Copilot follow-through**
16. If Gemini CLI supports safe stdin/file prompt transport, apply parity transport fix; otherwise keep strict guard + actionable error.
17. Add Copilot streaming usage extraction if the provider emits it; otherwise explicitly mark it as estimated-only.

#### Implementation Notes
- Keep model metadata intact; the `200k` cap is policy, not a change to model specs.
- Do not depend on provider-native compaction signals for correctness.
- Keep compaction provider-agnostic and deterministic from the app’s perspective.
- Treat transport safety and context safety as separate checks even when both happen in the same preflight service.

#### Rollout Thresholds

| Condition | Action | Notes |
|-----------|--------|-------|
| `< 70%` of effective cap | `send` | No user intervention |
| `70% - <85%` | `warn` | Non-blocking warning; no compaction yet |
| `85% - <92%` | `compact` | Auto-compact before send |
| `>= 92%` | `compact` or `block` | Compact first; block only if compaction cannot reduce enough |
| transport budget exceeded | `block` | Transport safety beats context safety |

#### Canonical Usage Shape

All persisted chat usage should converge on:

```typescript
interface CanonicalTokenUsage {
  promptTokens: number;
  completionTokens: number;
  totalTokens: number;
}
```

Compatibility rules:
- Persist new data in canonical camelCase only.
- Read legacy snake_case fields for older messages.
- Never assume provider-reported usage exists.
- Budgeting layer must know whether the source is exact, provider-reported, estimated, or opaque.
