## Summary

Replace fake tool schemas in the eval harness with a headless backend instance of the real Nexus agent stack. Build TestVault (filesystem-backed Obsidian Vault mock), TestApp, and HeadlessAgentStack factory. Initialize real ContentManager, StorageManager, CanvasManager, SearchManager, and ToolManager against a test vault directory. Real `getTools` returns real schemas, real `useTools` routes to real tools operating on real files. ~400-500 lines.

## What to Build

### 1. TestVault (~200-300 lines)
Filesystem-backed implementation of the Obsidian `Vault` interface. Tools use `app.vault` directly, so this is the critical piece.

**Must implement:**
- `getFileByPath()` / `getFolderByPath()` / `getAbstractFileByPath()` — return TFile/TFolder
- `getMarkdownFiles()` / `getRoot()` / `getName()` — for agent descriptions
- `read()` / `cachedRead()` / `modify()` / `create()` / `createFolder()` / `rename()` — file operations
- `adapter.exists()` / `adapter.stat()` / `adapter.read()` / `adapter.write()` / `adapter.mkdir()` / `adapter.list()` — raw filesystem adapter

**Key insight**: Tools use two patterns — `vault.*` methods (high-level) and `vault.adapter.*` methods (low-level). Both need to work consistently with an in-memory TFile/TFolder registry tracking what files exist.

### 2. TestApp (~50 lines)
Wraps TestVault + minimal stubs:
- `vault: TestVault`
- `fileManager: { trashFile: ... }` (delete = remove file)
- `metadataCache: { getFileCache: ... }` (return null or minimal cache)
- `workspace: {}` (stub)
- `plugins: { getPlugin: () => null }` (SearchManager handles null gracefully)

### 3. HeadlessAgentStack (~50-80 lines)
Factory function:
```typescript
async function createHeadlessAgentStack(testVaultPath: string): Promise<{
  app: TestApp;
  agentRegistry: Map<string, IAgent>;
  toolManager: ToolManagerAgent;
  getTools: (params) => Promise<GetToolsResult>;
  useTools: (params) => Promise<UseToolResult>;
}>
```
- Creates TestApp pointed at testVaultPath
- Initializes ContentManager, StorageManager, CanvasManager, SearchManager
- Creates ToolManager with the agent registry
- Returns the tool execution interface

### 4. TestVaultManager (~50 lines)
Reset/seed the test vault between scenarios:
- `reset()` — clear all files
- `seed(files: Record<string, string>)` — write initial files from fixture
- `snapshot()` / `restore()` — for scenario isolation

## Initialization Order

1. Create temp directory for test vault
2. Instantiate TestVault and TestApp
3. Instantiate ContentManagerAgent(testApp), StorageManagerAgent(testApp), CanvasManagerAgent(testApp)
4. Instantiate SearchManagerAgent(testApp, false) — vector modes off
5. Build agent registry Map
6. Instantiate ToolManagerAgent(testApp, registry, { workspaces: [], customAgents: [], vaultRoot: [] })
7. Export getTools/useTools for the eval harness

## Known Blockers / Open Questions

| Issue | Severity | Notes |
|-------|----------|-------|
| `app.metadataCache.getFileCache()` | HIGH | ContentManager's setProperty tool uses this for frontmatter. Mock to return parsed YAML or skip setProperty initially |
| TFile/TFolder in-memory tracking | MEDIUM | TestVault needs consistent file tree that updates on create/modify/rename/delete. Stat tracking (mtime, size) matters |
| `document.createElement` in SearchManager | LOW | `createMinimalPlugin()` uses DOM APIs. Jest/jsdom likely handles this |
| StorageManager `open` tool | LOW | Calls `app.workspace.getLeaf()` — inherently UI. Stub to return success |
| `getNexusPlugin()` in ToolBatchExecutionService | LOW | Returns null → workspace validation passes (fallback behavior) |
