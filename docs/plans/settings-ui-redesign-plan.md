## Summary

Modernize all settings card/form views across the Claudesidian MCP plugin for visual consistency, native Obsidian feel, cross-platform correctness, and mathematically-grounded spacing. This covers all tabs using the shared card system (Workspaces, Providers, Apps, Prompts), all nested views (projects, tasks, workflow editor, file picker), chat settings modals, and introduces a CSS spacing token system, search/filter on all card list views, and a simplified back-button navigation pattern. The approach is incremental — 4 waves, each independently testable and committable.

#### Design Approach

**Composition over inheritance** for search: `SearchableCardManager` wraps `CardManager` rather than extending it.

```
SearchableCardManager<T>
  ├── search input (TextComponent, shown when items > 5)
  ├── optional group headers (for Providers: Local/Cloud, Apps: Installed/Available)
  └── CardManager<T> (delegates to existing, receives filtered items)
```

**Filter logic extracted as pure function**: `filterItems(items: T[], query: string, filterFn): T[]` — independently testable, no DOM dependency.

**Navigation: BreadcrumbNav (kept) + BackButton (new)**:
- **BreadcrumbNav** stays for WorkspacesTab which has real depth (Workspaces → Detail → Workflow/FilePicker/Projects → Project Detail → Task Detail). Modernized with chevron icon separators (`setIcon(span, 'chevron-right')`) instead of "/" text, spacing tokens, and Obsidian CSS vars.
- **BackButton** (new) for simple 2-level views like PromptsTab where breadcrumbs are overkill. `setIcon(btn, 'arrow-left')` + parent label, current page title becomes `<h3>` below.

**CSS token system**: Defined globally at `:root` for reuse, but migration scoped to settings-related selectors only (~200-300 rules). Chat/modal styles remain unchanged in this pass.

#### Key Decisions

| Decision | Options | Resolution | Rationale |
|----------|---------|------------|-----------|
| Search implementation | A) Extend CardManager B) Composition wrapper | B) SearchableCardManager | SRP preserved; handles grouped views (Providers/Apps); filter logic extractable for testing |
| WorkspacesTab decomposition | A) Decompose first B) Defer | B) Defer to separate ticket | Keeps scope manageable; surgical changes don't require decomposition |
| CSS class rename (`agent-management-*`) | A) Rename now B) Defer | B) Defer | Cosmetic churn, increases diff size, zero functional value |
| Form abstraction | A) FormBuilder B) Direct components | B) Direct Obsidian components | Forms differ too much for one-size-fits-all; standardize CSS classes instead |
| ProvidersTab/AppsTab unification | A) Migrate to CardManager B) Keep direct Card | A) Migrate via SearchableCardManager | Consistency; search for free; grouped sections handled by wrapper |
| 6px spacing values | Map to 4px or 8px | Case-by-case: tight gaps → 4px, element spacing → 8px | Visual inspection during implementation |
| Border-radius values | Hardcoded or Obsidian vars | Obsidian vars (`--radius-s/m/l`) | Theme compatibility, cross-platform consistency |

#### Interface Contracts

```typescript
// Search filter contract — pure function, independently testable
interface CardSearchConfig<T extends CardItem> {
  placeholder: string;
  filterFn: (item: T, query: string) => boolean;
  minItemsForSearch?: number; // default: 5
}

// SearchableCardManager config — extends CardManager's config
interface SearchableCardManagerConfig<T extends CardItem> extends CardManagerConfig<T> {
  search?: CardSearchConfig<T>;
  groups?: CardGroup<T>[]; // for grouped views (Providers, Apps)
}

// Card group contract (for Providers/Apps pattern)
interface CardGroup<T extends CardItem> {
  title: string;        // e.g., "LOCAL PROVIDERS", "INSTALLED APPS"
  items: T[];
}

// Back button contract — simpler than BreadcrumbNavItem[]
interface BackButtonConfig {
  label: string;       // parent label text (e.g., "Workspaces")
  onClick: () => void; // navigation callback
}
```
