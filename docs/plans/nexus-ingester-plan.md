## Summary

The Nexus Ingester adds file ingestion to Nexus: users drag PDF or audio files onto the chat view (video deferred to v2), which processes them and creates a structured Obsidian note. Output notes open with `![[source-file]]` at the top so the original is embedded inline, followed by the extracted content. PDF processing offers two user-selectable modes: **pdf-lib** (default — text extraction, free, works for text-layer PDFs) or **Vision** (user picks a provider + model from capability-filtered dropdowns, for scanned/complex PDFs). Audio transcription uses OpenAI Whisper, Groq Whisper, and Google's multimodal pathway. Ollama and LM Studio support Vision OCR; neither supports audio transcription in v1 (no Whisper endpoint). Output notes are stored in a configurable output folder (default: `Ingested/`). Multiple dropped files are queued with stacked progress banners. A new `IngestAgent` exposes `ingest` and `listCapabilities` tools via the Two-Tool Architecture.

#### Design Approach

**Agent Design**: New `IngestAgent` in `src/agents/ingestManager/` (follows existing naming: taskManager, memoryManager, etc.). Does not extend ContentManager or StorageManager — different dependency profile (needs LLMProviderManager) and different SRP (file processing, not note editing or file management).

**Service Architecture**:
```
IngestTool.execute(params)
  → FileTypeDetector.detect() → { type, mimeType }
  → IngestionPipelineService.process()
      ├─ [PDF — text mode (default)] PdfTextExtractor.extract() via pdf-parse → text per page
      ├─ [PDF — vision mode] OcrService.process()
      │     ├─ PdfPageRenderer.renderPages() (pdfjs-dist → canvas → base64 PNG)
      │     ├─ VisionMessageFormatter.format() per provider
      │     └─ adapter.generate() with image content
      └─ [audio] TranscriptionService.transcribe()
            ├─ AudioChunkingService.chunk() (Web Audio API)
            ├─ MultipartFormDataBuilder.build()
            └─ ProviderHttpClient.request() to /v1/audio/transcriptions
  → OutputNoteBuilder.build() → vault.create()
```

**Provider Capability Pattern**:
- OCR: providers with `supportsImages: true` (existing flag) + Ollama vision models
- Transcription: new `supportsAudioTranscription` flag; only OpenAI, Groq (Whisper), Google (multimodal)
- Ollama/LM Studio: OCR only in v1 (no Whisper endpoint)

**Vision Message Format Strategies** (4 variants):
1. **OpenAI-style** (OpenAI, Groq, OpenRouter, LM Studio, Mistral, Copilot): `image_url` content part
2. **Anthropic-style**: `type: "image"` with `source.type: "base64"`
3. **Google-style**: `inline_data` with `mime_type` + `data`
4. **Ollama-style**: `images: [base64_string]` array directly on message object (unique format)

**Settings Integration**: Follow existing `defaultImageModel` pattern exactly:
```typescript
// Added to LLMProviderSettings (or a new top-level `ingestion` key on MCPSettings):
defaultPdfMode?: 'text' | 'vision';  // default: 'text'
defaultOcrModel?: { provider: string; model: string };  // used when defaultPdfMode = 'vision'
defaultTranscriptionModel?: { provider: string; model: string };
```

**Output Note Format**:

File flow:
- Original file `subfolder/report.pdf` — **stays in place**
- Markdown note **created** at `subfolder/report.md` — alongside the original

Note structure (no frontmatter — user decides their own organisation):
```
![[report.pdf]]

## Page 1
[extracted content]

## Page 2
...
```

For audio:
```
![[recording.mp3]]

[00:00:01] Hello and welcome...
[00:00:15] Today we're going to discuss...
```

Notes:
- Both files live in the same folder; `![[report.pdf]]` always resolves via Obsidian's normal link management and stays valid if the note is moved
- `vault.create()` used for the output `.md` note
- Body: page sections (`## Page 1`) for OCR, timestamped segments (`[00:01:23] text...`) for transcription

#### Key Decisions

| Decision | Options | Recommendation | Rationale |
|----------|---------|----------------|-----------|
| Agent location | New IngestAgent vs extend ContentManager/StorageManager | New IngestAgent | SRP, different deps (needs LLMProviderManager), ContentManager/StorageManager have no LLM deps |
| OCR/transcription services | Unified FileIngestionService vs separate services | Separate OcrService + TranscriptionService | Fundamentally different APIs (vision vs audio); different provider coverage; independently testable |
| Adapter approach | New adapters vs extend existing | Extend OllamaAdapter + LMStudioAdapter `buildMessages()` | Adapters already exist with ProviderHttpClient, localhost exemption, model management |
| Processing pipeline | Synchronous vs async queue | Async with progress callback | Large files require non-blocking; page-by-page processing inherently chunked |
| PDF rendering library | pdfjs-dist vs pdf-parse | pdfjs-dist for Vision mode; pdf-parse for text-extraction mode | pdfjs-dist renders to images (needed for Vision); pdf-parse extracts text layer (cheap default) |
| Audio chunking | Web Audio API vs ffmpeg | Web Audio API | No new deps; Electron Chromium has full codec support for common formats |
| Settings UI location | New Defaults tab section vs Providers tab | New section in Defaults tab | OCR/transcription defaults are per-task preferences, not provider configuration |
| Mobile support | Skip drag-and-drop vs file-menu | File-menu on all platforms, drag-and-drop desktop only | `Platform.isDesktop` guard for drag events; file context menu works mobile |
| Ollama transcription | Support (experimental) vs skip v1 | Skip v1 | No Whisper-compatible endpoint confirmed by preparer + backend |
| Video support | v1 vs v2 | **Deferred to v2** | Reduces scope; audio covers primary use cases; Web Audio pathway clear for future |
| PDF default mode | pdf-lib text extraction vs Vision LLM | **User-selectable; pdf-lib default** | Default = pdf-parse text extraction (free, no API); Vision = provider/model dropdown filtered to vision-capable providers |
| pdfjs-dist bundle size | Accept now vs validate first | **Validate in PREPARE** | Prototype inline worker, measure tree-shaken size; if >3MB find lighter alternative |
| Batch file drops | Single vs queue | **Queue multiple files** | Queue architecture handles naturally; stacked progress banners in UI |

#### Interface Contracts

```typescript
// Core types — src/agents/ingestManager/types.ts
interface IngestParams {
  filePath: string;
  outputPath?: string;
  ocrProvider?: string;
  ocrModel?: string;
  transcriptionProvider?: string;
  transcriptionModel?: string;
  systemPrompt?: string;
  concurrency?: number; // default 3
  timestampFormat?: '[HH:MM:SS]' | '[Xm Ys]' | '[H:MM:SS.s]'; // default [HH:MM:SS]
}

interface IngestResult {
  success: boolean;
  outputPath?: string;
  pageCount?: number;
  durationSeconds?: number;
  processingTimeMs: number;
  tokenUsage?: TokenUsage;
  warnings?: string[];
  error?: string;
}

interface IngestProgress {
  phase: 'detecting' | 'rendering' | 'processing' | 'writing';
  current: number;
  total: number;
  message: string;
}

// Provider capability interface — wraps existing adapter
interface IOcrProvider { extractText(imageBase64: string, mimeType: string, prompt?: string): Promise<string>; }
interface ITranscriptionProvider { transcribe(audioBuffer: ArrayBuffer, mimeType: string, options: TranscriptionOptions): Promise<TranscriptionSegment[]>; }

interface TranscriptionSegment { startSeconds: number; endSeconds: number; text: string; }
```

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| pdfjs-dist worker fails to load in Electron | Medium | High | PREPARE prototype; fallback: pdf-parse text-only mode |
| Google Gemini 15 RPM limit blocks large PDFs | High | Medium | Per-provider concurrency defaults (Google: 1 page/2s); user warning |
| Memory pressure from large PDFs | Medium | Medium | Process-and-discard; max concurrency 3; hard limit at 100 pages with warning |
| Ollama/LM Studio not running at ingest time | High | Low | Clear error: "Start Ollama at localhost:11434 to use OCR" |
| pdfjs-dist bundle bloat | Medium | Medium | Lazy-load; validate size in PREPARE |
| Audio codec not decodable by Web Audio | Low | Low | Catch decode error; surface supported formats list |
| Multi-page PDF rate limit on low-tier API keys | Medium | Low | Configurable concurrency; exponential backoff (existing withRetry()) |
