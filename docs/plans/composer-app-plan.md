## Summary

Add a new "Composer" app agent to Claudesidian MCP that stitches together multiple vault files of the same type into a single output file. Phase 1 supports markdown concatenation (zero dependencies) and PDF merging (via `pdf-lib`). Phase 2 adds audio merging (WAV output via Web Audio API/Crunker). Video merging is deferred to v2 due to GPL licensing and 62MB bundle concerns with ffmpeg.wasm. The Composer fits the existing BaseAppAgent / AppManager / two-tool (getTools/useTools) architecture with `credentials: []`.

#### Design Approach
Single `compose` tool with `format` enum dispatching to `IFormatComposer` strategy implementations. Platform gating at runtime (`Platform.isDesktop`) to allow markdown/PDF on mobile while restricting audio to desktop — do NOT use `manifest.isDesktopOnly: true` (that would block all formats on mobile). All-or-nothing error model: partial composition fails with `failedFiles[]` array for LLM self-correction. Credential-less App Agent pattern with empty `credentials: []`.

#### Key Decisions
| Decision | Options | Recommendation | Rationale |
|----------|---------|----------------|-----------|
| Tool granularity | Single `compose` vs. 4 per-type tools | Single `compose` with `format` param | Uniform API, simpler LLM usage, easier to add formats |
| Platform gating | `manifest.isDesktopOnly` vs runtime check | Runtime `Platform.isDesktop` check | Markdown/PDF work on mobile; only audio/video need desktop |
| Error model | All-or-nothing vs partial success | All-or-nothing with `failedFiles[]` | Partial composition is semantically meaningless |
| Agent type | BaseAppAgent vs plain BaseAgent | BaseAppAgent with `credentials: []` | Consistent App pattern, extensible for future cloud rendering |
| Audio output format | WAV only vs multi-format | **WAV + WebM/Opus + MP3** | WAV (native), WebM/Opus (MediaRecorder), MP3 (wasm-media-encoders MIT) |
| Audio encode strategy | lamejs (LGPL) vs wasm-media-encoders (MIT) vs MediaRecorder | **wasm-media-encoders + MediaRecorder** | Avoids LGPL entirely; wasm-media-encoders for MP3, MediaRecorder for WebM/Opus |
| Audio operation modes | Concat only vs concat + mix/layer | **Both: concat + mix** | User wants dialogue + SFX + ambient layering; `OfflineAudioContext` handles both |
| Video v1 scope | Include (ffmpeg.wasm GPL-2.0) vs defer | **Defer to v2** | GPL-2.0 license blocker + 62MB WASM core unacceptable |
| Input constraint | Same-type only vs mixed types | **Same-type only** | Mixed-type output semantics are undefined |

#### Interface Contracts
```typescript
interface IFormatComposer {
  compose(files: TFile[], vault: Vault, options: ComposeOptions): Promise<Uint8Array | string>;
  supportedExtensions: string[];
  isAvailableOnPlatform: boolean;
}

// Tool input schema
interface ComposeParams {
  files: string[];              // Vault-relative paths (same type)
  format: 'markdown' | 'pdf' | 'audio' | 'video'; // video = stub ("not yet supported") until Phase 3
  outputPath: string;           // Vault-relative output path
  separator?: string;           // Markdown: section separator (default: '\n---\n')
  includeFileHeaders?: boolean; // Markdown/text: prepend filename as H2 heading
  maxFileSizeMb?: number;       // Per-file size limit override (default: 50)
  outputFormat?: 'wav' | 'mp3' | 'webm'; // Audio only; default: 'wav'

  // Audio mix/layer mode (audioMode: 'mix' only — replaces flat files[] for audio)
  audioMode?: 'concat' | 'mix';       // default: 'concat'
  tracks?: Array<{                     // Used when audioMode = 'mix'
    file: string;                      // Vault-relative path
    volume?: number;                   // 0.0–1.0, default: 1.0
    offset?: number;                   // Start time in seconds, default: 0
    fadeIn?: number;                   // Fade-in duration in seconds, default: 0
    fadeOut?: number;                  // Fade-out duration in seconds, default: 0
  }>;
  duration?: number;                   // Mix mode: total output duration (seconds); default: longest track
}

// Tool result
type ComposeResult =
  | { success: true; path: string; fileCount: number; totalInputSize: number; outputSize: number; }
  | { success: false; error: string; failedFiles?: string[]; }
```

## Open Questions

### Require User Decision
- [x] **Bundle size**: ✅ RESOLVED — ~10MB total bundle is acceptable. pdf-lib included in Phase 1.

- [x] **Video scope**: ✅ RESOLVED — Deferred to v2. Composer v1 is markdown + PDF + audio only.

- [x] **Mixed inputs**: ✅ RESOLVED (default accepted) — Same-type only per call. Mixed-type output semantics are unclear.

- [x] **Audio output format**: ✅ RESOLVED — Multi-format via tiered approach:
  - **WAV**: Direct from `OfflineAudioContext` AudioBuffer — zero deps
  - **WebM/Opus**: MediaRecorder API (`audio/webm;codecs=opus`) — Chromium-native, MIT codec, excellent compression, no extra deps
  - **MP3**: `wasm-media-encoders` (MIT-licensed WASM, ~200KB) — avoids LGPL lamejs
  - Note: MediaRecorder cannot encode to MP3 natively in Chromium — that's why `wasm-media-encoders` is needed for MP3 specifically
  - ✅ Verified: `wasm-media-encoders` MIT confirmed, ~200KB, Electron compatible, external WASM via existing esbuild `.wasm: "file"` pattern

### Require Further Research
- [x] **MediaRecorder + audio encoding**: ✅ VERIFIED — `audio/webm;codecs=opus` confirmed in Electron; MP3 requires `wasm-media-encoders`; no other MIT MP3 encoder exists
- [x] **OfflineAudioContext mobile**: ✅ VERIFIED — Available in Electron AND mobile WebView; works fully offline; no Electron-specific issues
- [ ] **pdf-lib tree-shaking**: Confirm actual bundled size with esbuild (standard-fonts not needed for PDF merge — may reduce below 1.5MB)
